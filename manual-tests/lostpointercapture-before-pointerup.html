pointerup retargets off the captured element

Repro — macOS Chromium, intermittent; repeat several times. Press the grey box (turns green, captures the pointer), drag off it, and release. Two independent cases, by where you release:

  • Inside the window — document pointerup target=BODY.
  • Outside the window, or onto the scrollbar — document pointerup target=HTML.

Pass (either): the box turns grey — its own pointerup ran. Fail: the box stays green — capture was released first, the pointerup retargeted off the box, and only the document listener saw it.

Spec order is pointerup at the box, then lostpointercapture (Pointer Events — Implicit release of pointer capture). The bug reverses them.
