L AIC Chain tab: High-Level Architecture
========================================

Scope
-----
Architecture of the L AIC Chain tab, in scripts/aic-chain.js.
Renders the muscle chain panel, an SVG overlay of anchor circles
on the body image, leader lines from row to circle, and a detail
panel for the active muscle. Single active-muscle selection model.

Reference: layering doctrine in prd/architecture/layering.md.
Sibling: prd/hla/diagnose.txt (causal chains).

Pillar mapping
--------------
Module code  - scripts/aic-chain.js (init, resolveDomRefs, buildPanel,
               setActiveMuscle, drawLeaderLine, getOrCreate*View,
               handleResize, attachListeners). Co-located with the ADT.
ADT          - AicMuscle (per-muscle decorator) + AicMuscleFactory
               (chain-level data vendor + lazy instance cache).
               Co-located in scripts/aic-chain.js.
Data         - data/aic-chain.json. Top-level keys: chain, detail,
               fieldLabels. fieldLabels keys (role, pattern,
               correction) drive detail-panel row construction.
Loader       - direct fetch('data/aic-chain.json') inside init.
               Bad-data boundary at fetch/parse only; no schema
               validation (file is shipped with the app).


Init
----
init() ---+--> fetch('data/aic-chain.json')
          |    bad path: showFetchError -> return
          +--> AicMuscleFactory.acceptData(data)
          |       (clears instance cache, stores chain/detail/labels)
          +--> resetView()  (clears panel, overlay, leader, detail)
          +--> buildPanel() iterates Factory.forEachEntry:
          |       row(id, chainEntry) + optional connection note
          |       + optional terminus note (data-driven)
          +--> setupStaticUi() (idempotent: listeners + resize handle
                                + window/subtab-shown listeners)


User input -> Decorator method
------------------------------
+----------------------------------------------------------------+
|  USER INPUT                                                    |
|    click on .aic-chain-row --> panel delegation                |
|                                                                |
|       id     := rowEl.id                                       |
|       muscle := AicMuscleFactory.getInstance(id)               |
|       setActiveMuscle(muscle):                                 |
|         - if activeMuscle: activeMuscle.deactivate()           |
|         - muscle.mount(overlayEl)   (idempotent)               |
|         - muscle.activate()                                    |
|         - drawLeaderLine(muscle)                               |
|         - showDetail(muscle)                                   |
+----------------------------------------------------------------+


AicMuscleFactory (static class, module-private state)
-----------------------------------------------------
+-- #instances    Map<id, AicMuscle>     (lazy cache)
+-- #chainData    { id -> chainEntry }   (label, priColor, anchor,
|                                         connection?, terminus?)
+-- #detailData   { id -> detailEntry }  (role, pattern, correction)
+-- #fieldLabels  { key -> label }       (drives detail row order)
|
+-- acceptData(data)        seed all four; clear instance cache
+-- getInstance(id)          pure: cache hit, else construct + cache
+-- forEachEntry(callback)   iterate chain entries (panel build)
+-- fieldLabelEntries()      [[key,label], ...] (detail-panel build)


AicMuscle (per-muscle decorator)
--------------------------------
+-- private  #id #chainEntry #detailEntry
|            #rowEl #circlesByView (Map<view, SVGCircle>) #mounted
+-- public   label() priColor() rowEl() anchor(view)
|            hasDetail() field(key)
|            mount(overlayEl)   idempotent: resolves rowEl,
|                               builds anterior+posterior circles,
|                               appends to overlay
|            activate()         add .activeMuscle to row + circles
|            deactivate()       remove .activeMuscle
+-- effects  DOM mutations on its own row + its circles only


Module orchestration layer
--------------------------
+-- DOM regions     containerEl, panelEl, overlayEl, leaderEl,
|                   imgEl, detailEl, imageColEl, tabSectionEl,
|                   subtabContentEl  (resolved once in
|                   resolveDomRefs)
+-- Active Object   activeMuscle (single AicMuscle reference)
+-- Lazy views      detailView (cached panel + valueElsByKey),
|                   leaderView (cached SVG path)
+-- Idempotent      isStaticUiReady gates listener/resize binding
|                   so re-init does not double-bind
+-- Templates       detailFieldRowTemplate (lazy, cloneNode for rows)


Coupling channels
-----------------
+-- id          DOM <-> JSON entry <-> Factory pool   (one string,
|               three layers; the JSON entries are held by the
|               Factory's static #chainData / #detailData)
+-- key         fieldLabels key <-> detailEntry field <-> valueElsByKey
|               (drives detail-panel row construction generically)
+-- closure     init() and module locals hold DOM region refs;
|               attachListeners closes over panelEl + the dispatch


Doctrine alignment
------------------
Shared Key       - id is the only string crossing DOM, DATA, and pool.
                   fieldLabels key is the only string crossing labels,
                   detail data, and rendered rows.
Decorator Factory- AicMuscleFactory.getInstance lazily caches AicMuscle
                   decorators per id; precedent is CausalChainFactory.
Encapsulation    - AicMuscle private state (#chainEntry, #detailEntry,
                   #circlesByView, #mounted); only id-derived accessors
                   and lifecycle methods are public.
DOM-Light        - SVG overlay attributes (viewBox, preserveAspectRatio)
                   declared in index.html; circle r/state in CSS.
CSS over JS for  - .activeMuscle class drives row + circle highlight;
state            - leader-line transition lives in CSS.
                 - .aic-chain-connection arrow via ::before, scoped
                   :not(.aic-chain-terminal).
Native First     - panel rows are simple <div>s; click delegation on
                   panelEl reaches any row via .closest('.aic-chain-row').
Active Object    - activeMuscle holds the single live decorator; no
                   querySelectorAll scans to clear .activeMuscle state.
Bad-data         - boundary lives only at fetch/parse: HTTP non-OK or
boundary           fetch/parse throw -> showFetchError. No schema
                   validation; data is shipped with the app.
