L AIC Chain tab: High-Level Architecture
========================================

Scope
-----
Architecture of the L AIC Chain tab, in scripts/aic-chain.js.
Renders the muscle chain panel, an SVG overlay of anchors on the body
image, leader lines from each row to its anchor, and a detail panel for
the active muscle. Single active-muscle selection model.

Reference: layering doctrine in prd/architecture/layering.md.
Related: prd/architecture/navigation-tabs.md for lazy import, module-top
setup, and ResizeObserver activation contracts.

Pillar mapping
--------------
Module code  - scripts/aic-chain.js (module-top bootstrap,
               resolveDomRefs, ensureLeaderDefs, buildPanel, setupUi,
               setActiveMuscle, drawLeaderLine, showDetail).
               Co-located with the ADT.
ADT          - AicMuscle. One class fuses the per-muscle decorator and
               the chain-level data + lazy instance pool.
               Static facet: data acceptance, instance pool, iteration
               helpers. Instance facet: per-muscle DOM and behavior.
               Symbol-guarded constructor — only AicMuscle.getInstance
               can call `new AicMuscle(...)`.
Data         - data/aic-chain.json. Top-level keys: chain, detail,
               fieldLabels. fieldLabels keys (role, pattern,
               correction) drive detail-panel row construction.
Loader       - loadJson('./data/aic-chain.json'), run at module top
               through attemptLoad: it renders on success and shows a
               retry callout in the container on failure. No schema
               validation; the file ships with the app.


Module evaluation
-----------------
import aic-chain.js
          +--> containerEl := querySelector('.aic-chain-container'),
          |    then resolveDomRefs() resolves the required descendants.
          |    missing descendant: appendErrorCallout -> stop before
          |    data load
          +--> attemptLoad(loadJson('./data/aic-chain.json'))
          |    failure: retry callout -> stop. success: render callback:
          +--> AicMuscle.acceptData(data)
          |       (destructures chain/detail/fieldLabels into statics)
          +--> buildPanel() iterates AicMuscle.forEachEntry:
          |       row(id, chainEntry) + optional connection note
          |       + optional terminus note (data-driven)
          +--> setupUi() runs ensureLeaderDefs() (idempotent — inserts
                         <defs><marker> into the leader SVG via
                         insertAdjacentHTML), then binds panel click
                         delegation, resize handle, panel scroll, and
                         ResizeObserver(tabSectionEl) -> drawLeaderLine.

Navigation-tabs lazy-loads this file for anatomy-aic-content. The import
promise is the readiness signal. There is no exported startup function.
ES module caching makes setupUi run once per page load.


User input -> behavior method
-----------------------------
+----------------------------------------------------------------+
|  USER INPUT                                                    |
|    click on .aic-chain-row --> panel delegation                |
|                                                                |
|       id     := rowEl.id                                       |
|       muscle := AicMuscle.getInstance(id)                      |
|       setActiveMuscle(muscle):                                 |
|         - if muscle === activeMuscle: return (no-op)           |
|         - if activeMuscle: activeMuscle.deactivate()           |
|         - muscle.mount()            (idempotent;               |
|                                      reads module overlayEl    |
|                                      and leaderEl)             |
|         - muscle.activate()                                    |
|         - drawLeaderLine({animate: true})                      |
|                                      (rAF-coalesced; reads     |
|                                      activeMuscle inside the   |
|                                      rAF, so back-to-back      |
|                                      clicks always render the  |
|                                      latest selection)         |
|         - showDetail(muscle)                                   |
+----------------------------------------------------------------+


AicMuscle: static facet (chain-level data + lazy instance pool)
---------------------------------------------------------------
+-- #instances    { id: AicMuscle }      (lazy cache, Object.create(null);
|                                         initialized once at class load)
+-- #chainData    { id -> chainEntry }   (label, priColor, anchor,
|                                         connection?, terminus?)
+-- #detailData   { id -> detailEntry }  (per-field values; fieldLabels
|                                         keys select what renders)
+-- #fieldLabels  { key -> label }       (drives detail row order)
+-- #KEY          Symbol                 (constructor guard token)
+-- #toXY         ([x, y]) => ({x, y})   (data tuple -> point object)
|
+-- acceptData(data)         destructure chain/detail/fieldLabels
+-- getInstance(id)          cache hit, else `new AicMuscle(id, ..., #KEY)`
+-- forEachEntry(callback)   iterate chain entries for panel build
+-- fieldLabelEntries()      [[key,label], ...] for detail panel


AicMuscle: instance facet (per-muscle decorator)
------------------------------------------------
+-- private  #id #chainEntry #detailEntry
|            #rowEl
|            #anchorsByView      Map<view, SVG <g>>      (overlay)
|            #leaderPathsByView  Map<view, SVGPath>       (leader SVG)
|            #mounted
+-- constructor(id, chainEntry, detailEntry, key)
|     throws if key !== AicMuscle.#KEY (external `new` blocked)
|     throws if chainEntry is falsy (unknown id; named in the error)
+-- public   label() priColor() rowEl() anchor(view)
|            hasDetail() field(key)
|            anchor(view)        returns {x, y} (data tuple converted
|                                via static #toXY)
|            mount()             idempotent: resolves rowEl from id,
|                                builds anterior+posterior anchor groups
|                                into module overlayEl, builds anterior+
|                                posterior leader paths into module
|                                leaderEl. Path classList carries the
|                                muscle's priColor; the stroke color comes
|                                from CSS (.aic-leader-path { stroke:
|                                var(--pri-fg) }), and the arrowhead
|                                marker uses fill="context-stroke" so
|                                it matches automatically.
|            activate()          add .activeMuscle to row + anchor groups;
|                                scroll row into view
|            deactivate()        remove .activeMuscle from row + anchor
|                                groups; clear `d` attribute on this
|                                muscle's leader paths (hides them — no
|                                display toggle, no class on paths)
|            drawLeader({start, imageRect, sectionRect, animate})
|                                for each view: compute the end point
|                                (anchorPointInSection), then
|                                drawLeaderPath sets the path d and, when
|                                animate is set, runs the Web Animations
|                                draw-on.
+-- effects  DOM mutations on its own row + its anchors + its leader
             paths only.


Module orchestration layer
--------------------------
+-- DOM regions     containerEl, panelEl, panelScrollEl, overlayEl,
|                   leaderEl, imgEl, detailEl, detailScrollEl,
|                   imageColEl, tabSectionEl
|                   (resolved once in resolveDomRefs; AicMuscle.mount
|                   reads overlayEl and leaderEl directly from the
|                   module scope)
+-- Active Object   activeMuscle (single AicMuscle reference)
+-- Detail panel    showDetail rebuilds it per selection
|                   (replaceChildren); skipped when the muscle has no
|                   detail entry
+-- Once-only setup panel listener, resize handle, ResizeObserver, panel
|                   scroll listener, and ensureLeaderDefs all wire in
|                   setupUi, which runs once per page load
+-- rAF coalescing  drawFrameId — module-scope handle. drawLeaderLine
|                   coalesces back-to-back triggers (rapid resize,
|                   scroll, click then resize) into one rAF tick. Inside
|                   the rAF: all four rects (sectionRect, rowRect,
|                   panelRect, imgRect) are read first — section/row/panel
|                   give the clamped start, imgRect the end points — then
|                   sizeLeaderSvg and the path writes run.
+-- DOM helpers     newSvg / newEl (el-create.js) build elements from an
|                   attrs object; mount uses newSvg for anchor groups and
|                   leader paths, the row builders use newEl.


Coupling channels
-----------------
+-- id          DOM row id  <->  AicMuscle.#chainData key  <->
|               AicMuscle.#instances key. One string, three lookups.
+-- key         fieldLabels key <-> detailEntry field <-> rendered detail
|               rows (drives detail-panel row construction generically)
+-- priColor    chainEntry.priColor names a CSS class on:
|               - the chain row (pri-X class sets --pri-fg, tokens.css)
|               - the anchor group (.aic-anchor-dot fills --pri-fg)
|               - each leader path (.aic-leader-path stroke is
|                 var(--pri-fg), the marker arrowhead uses
|                 fill="context-stroke" so it inherits the path's
|                 stroke automatically — no JS color writes)
+-- closure     module locals hold DOM region refs; setupUi closes over
|               panelEl and resize targets; AicMuscle reads module
|               overlayEl/leaderEl directly during mount


Layout redraw
-------------
ResizeObserver(tabSectionEl), the resize handle's onResize callback, and
the panel's scroll listener all reference drawLeaderLine directly — there
is no handleResize wrapper. drawLeaderLine guards on `!activeMuscle` and
bails before scheduling, so events that fire before any selection are
no-ops. rAF coalescing absorbs bursts (drag-resize, observer chains,
scroll) into one redraw per frame. `getBoundingClientRect()` returns zero
when tabs are display:none, so the activation/redraw signal is the
ResizeObserver fire when the subtab becomes visible.


Doctrine alignment
------------------
Shared Key       - id is the only string crossing DOM, DATA, and pool.
                   fieldLabels key is the only string crossing labels,
                   detail data, and rendered rows.
Decorator + pool - AicMuscle.getInstance lazily caches AicMuscle
                   decorators per id. Static facet (data + pool +
                   iteration) and instance facet (per-muscle DOM)
                   co-located in one class — same pattern as
                   BlankPanelSite in anatomize.js.
Symbol guard     - Constructor takes a private-symbol key argument.
                   Throws on mismatch (`new AicMuscle(...)` outside the
                   factory) or on unknown id (chainEntry falsy). Catches
                   programmer error at the source rather than as a later
                   TypeError on a downstream method call.
Encapsulation    - AicMuscle private state (#chainEntry, #detailEntry,
                   #anchorsByView, #leaderPathsByView, #mounted); only
                   id-derived accessors and lifecycle methods are public.
DOM-Light        - SVG container declared in index.html. <defs> +
                   <marker> are inserted once at setupUi via
                   insertAdjacentHTML and a template literal — no
                   element-by-element setAttribute scaffolding.
                   Anchor groups and per-muscle leader paths are
                   constructed via newSvg in AicMuscle.mount.
CSS over JS for  - .activeMuscle class drives row + anchor highlight.
state              Leader stroke colors come entirely from CSS:
                   path's pri-X class sets --pri-fg via tokens.css;
                   .aic-leader-path uses stroke: var(--pri-fg); marker
                   uses fill="context-stroke". No resolveColor in JS.
                 - Leader draw-on is a Web Animations call: drawLeaderPath
                   sets the path d and strokeDasharray to the path length,
                   then animates strokeDashoffset to 0. Resize and scroll
                   redraws reposition without replaying it.
                 - .aic-chain-connection arrow via ::before, scoped
                   :not(.aic-chain-terminal).
Read/Write batch - drawLeaderLine's rAF reads every rect up front
                   (sectionRect, rowRect, panelRect, imgRect), then
                   writes: sizeLeaderSvg's viewBox, then the per-view
                   path d's in muscle.drawLeader. No DOM read after any
                   write. Pattern memory: read-write-batching-raf.md.
Native First     - Panel rows are <div>s; click delegation on panelEl
                   reaches any row via .closest('.aic-chain-row').
Active Object    - activeMuscle holds the single live decorator; no
                   querySelectorAll scans to clear .activeMuscle state.
Data boundary    - loadJson catches network, HTTP, and JSON parse
                   failures into a POJO result. attemptLoad branches on
                   result.ok, rendering on success and showing a retry
                   callout on failure. Render bugs are not caught as data
                   errors. No schema validation; data ships with the app.
