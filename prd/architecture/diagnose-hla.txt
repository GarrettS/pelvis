Diagnose tab — Causal Chains: High-Level Architecture
=====================================================

Scope
-----
Architecture of the causal-chain section, now in
scripts/diagnose-causal-chains.js after #22 split the diagnose tab
into per-feature modules. Built on #13 (untangle) and #17 (drag
refactor). The Shared-Key / factory / entity shape also describes
the case-study section (scripts/diagnose-case-studies.js), which
additionally exemplifies "decorator owns derived state" via
#correctTreatmentSet and isCorrectTreatment / isSelectedTreatment
query methods. Case studies do not share the drag-specific Active
Object fields, the document-level Esc handler, or the pointer
lifecycle.

Caveat: setupCausalChains() binds its listeners inline rather than
splitting one-shot setup from repeatable render. setupCausalChains
runs once per tab activation via navigation-tabs's lazy-init gate,
so this is latent in the success path. There is a known
retry-duplication path on partial-init failure (init throws after
some setup*() calls have bound listeners → navigation-tabs
releases the gate → revisit re-binds). Deferred; not a live bug
in normal flow.


Init
----
setupCausalChains() ─┬─► await getCausalChains() (cached slice)
                     ├─► attach listeners on #chains-wrap + document
                     ├─► populate CausalChainFactory pool
                     └─► render <ol class="chain-list" id="..."> + <li>s


User input → Decorator method
------------------------------
┌──────────────────────────────────────────────────────────────┐
│  USER INPUT                                                  │
│    click    ──┐                                              │
│    pointer* ──┼──► #chains-wrap delegation                   │
│    keydown  ──┼──► document delegation (Esc only)            │
│               │                                              │
│               ▼                                              │
│         closure dispatch                                     │
│           id     := target.closest('.chain-list').id         │
│           chain  := CausalChainFactory.getInstance(id)       │
│           chain.<method>(...)                                │
└──────────────────────────────────────────────────────────────┘


CausalChainFactory (closure)
----------------------------
├── #instances    { id -> CausalChain }
├── #KEY          Symbol (private-ctor invariant)
└── getInstance(elOrId, def?)
       id = elOrId.id || elOrId
       def ??= causalChains[id]   (local module-scope slice from
                                   getCausalChains() accessor)
       pool[id] ?? new CausalChain(id, def, KEY)


CausalChain (instance)
----------------------
├── private  #id #steps #order
│            #activeDragItem #dropTarget #dropTargetMarker
│            #startY #initialItemTop
├── public   startDrag  dragMove  commitDrop  endDrag
│            orderResults  isOrderCorrect  id (getter)
└── effects  DOM mutations (insertBefore, classList) on its <ol>


Coupling channels
-----------------
├── id            DOM <-> JSON slice <-> Factory pool   (one string,
│                 three layers; the JSON slice is held by the
│                 module's local causalChains, populated from
│                 getCausalChains() at setup time)
├── KEY           Factory <-> Decorator           (invariant guard)
└── closure       setupCausalChains() holds the listener wiring;
                  dispatch reaches Factory via lexical scope


Doctrine alignment
------------------
Shared Key       — id is the only string crossing DOM, DATA, and pool.
Encapsulation    — id is exposed; everything else stays private.
                   (Shared Key is a key-exposure rule, not a state-
                   exposure rule. Combine with Encapsulation; they
                   don't overlap.)
Event Delegation — single listener on #chains-wrap (and document for
                   Esc) covers all current and future descendants.
Native First     — <ol> + ::marker owns step numbering; reorder is
                   pure DOM-position; no JS sync layer.
Active Object    — #activeDragItem and #dropTargetMarker hold the
                   current actors directly; no querySelectorAll scan
                   to clear class state.
